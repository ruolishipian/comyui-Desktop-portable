# 配置文件保留方案说明

## 问题分析

覆盖安装时配置文件丢失的原因：
- 配置文件存储在应用安装目录下的 `config/` 文件夹
- NSIS 安装程序在覆盖安装时会先删除旧文件，导致配置丢失

## 解决方案

### 方案：NSIS 安装脚本自动备份恢复

**实现方式：**
在安装过程中自动备份和恢复配置文件，用户无需手动操作。

**修改内容：**

1. **创建 NSIS 脚本** (`scripts/installer.nsh`)
   - 安装前：自动备份 `config/`、`logs/`、`data/` 目录到临时目录
   - 安装后：自动恢复这些目录到安装位置
   - 清理临时备份

2. **更新 electron-builder.yml**
   - 添加 `include: "scripts/installer.nsh"` 引用自定义脚本

**优点：**
- ✅ 完全自动化，用户无感知
- ✅ 保持便携版特性（配置仍在应用目录）
- ✅ 支持所有类型的配置文件
- ✅ 安装过程显示备份和恢复状态

**缺点：**
- ⚠️ 需要额外的临时空间存储备份
- ⚠️ 如果安装失败，备份可能不会被清理（但会在系统重启时自动清理临时目录）

## 使用方法

### 打包安装程序

```bash
# 1. 编译代码
npm run build

# 2. 打包安装程序
npm run build:win:installer

# 3. 生成的安装包位于 dist/ 目录
# 文件名：ComfyUI-Desktop-portable-3.0.0-setup.exe
```

### 安装流程

1. 运行安装程序
2. 选择安装目录（可以是之前安装的目录）
3. 安装程序会自动：
   - 备份现有配置文件到临时目录
   - 安装新版本文件
   - 恢复配置文件
   - 清理临时备份
4. 启动应用，所有配置保持不变

## 备份的文件

安装程序会自动备份以下目录：
- `config/` - 配置文件（包括 `portable-config.json`）
- `logs/` - 日志文件
- `data/` - Electron 缓存和会话数据

## 注意事项

1. **首次安装**：没有配置文件，不会执行备份恢复操作
2. **覆盖安装**：自动备份恢复，配置不会丢失
3. **卸载**：配置文件不会被删除（`deleteAppDataOnUninstall: false`）
4. **临时目录**：备份存储在 Windows 临时目录（`%TEMP%\ComfyUI-Desktop-Backup`）

## 测试建议

1. 首次安装并配置应用
2. 记录配置内容（如 Python 路径、ComfyUI 路径等）
3. 使用新版本安装程序覆盖安装
4. 启动应用，验证配置是否保留

## 技术细节

### NSIS 脚本工作原理

```nsis
; 安装前（customInit）
1. 创建临时备份目录
2. 复制 config、logs、data 到临时目录

; 安装后（customInstall）
1. 从临时目录恢复 config、logs、data
2. 删除临时备份目录
```

### 兼容性

- ✅ Windows 7 及以上版本
- ✅ 支持 NSIS 3.x
- ✅ 兼容 electron-builder 26.x

## 其他方案对比

### 方案A：使用用户数据目录（已放弃）
- 配置存储在 `%APPDATA%\ComfyUI-Desktop-portable`
- ❌ 不符合便携版设计理念
- ❌ 无法实现"即插即用"

### 方案B：手动备份（不推荐）
- 用户手动复制 config 目录
- ❌ 用户体验差
- ❌ 容易忘记

### 方案C：NSIS 自动备份恢复（已采用）
- ✅ 自动化，用户无感知
- ✅ 保持便携版特性
- ✅ 可靠性高
