# 启动器可靠性改进方案

## 问题分析

根据日志分析，发现以下问题：

### 1. 启动超时误判
```
[01:26:11] [error] 服务器启动超时: Timed out waiting for: http://127.0.0.1:8188/queue, ...
[01:26:10] [info] Starting server
[01:26:10] [info] To see the GUI go to: http://127.0.0.1:8188
```
**问题**：ComfyUI 实际已启动成功，但 HTTP 端点检测超时导致误判为失败。

### 2. 插件加载时间过长
```
   39.6 seconds: D:\ai\ComfyUI\ComfyUI\custom_nodes\comfy-mtb
   10.5 seconds: D:\ai\ComfyUI\ComfyUI\custom_nodes\ComfyUI-Easy-Use
```
**问题**：部分插件加载时间超过 30 秒，导致默认超时时间不足。

### 3. 进程存活检测缺失
**问题**：启动失败时直接杀死进程，未检查进程是否实际存活。

## 改进方案

### 1. 渐进式检测机制

**原方案**：
- 一次性检测所有 HTTP 端点（`/queue`, `/`, `/system_stats`）
- 超时直接失败并杀死进程

**新方案**：
- **第一阶段**：快速检测基础端口（30秒超时）
  - 仅检测 `http://127.0.0.1:8188/`
  - 快速响应，快速失败
  
- **第二阶段**：完整端点检测（剩余时间）
  - 检测 `/queue` 和 `/system_stats`
  - 允许更长的等待时间

### 2. 进程存活检测

**新增逻辑**：
```typescript
const isProcessAlive = this._process !== null && !this._process.killed && this._process.pid !== undefined;

if (isProcessAlive) {
  // 进程还活着，给予宽容处理
  logger.warn('HTTP端点检测超时，但进程仍在运行');
  logger.warn('可能原因: 插件加载慢、某些自定义节点有问题、或端点响应慢');
  
  // 尝试基础端口检测
  // 如果可访问，标记为运行状态
  // 否则仍标记为运行并启动健康检查
}
```

**优点**：
- 避免误杀正在启动的进程
- 给予进程足够的启动时间
- 通过健康检查监控实际状态

### 3. 超时时间调整

**调整**：
- 默认超时：180秒 → **300秒（5分钟）**
- 快速检测：30秒
- 完整检测：270秒
- 检查间隔：1秒
- 窗口时间：2秒（确保服务稳定）

### 4. 失败处理改进

**原方案**：
- HTTP 端点超时 → 立即杀死进程 → 标记为失败

**新方案**：
1. HTTP 端点超时
2. 检查进程是否存活
3. 如果进程存活：
   - 尝试基础端口检测
   - 如果可访问 → 标记为运行状态
   - 如果不可访问 → 仍标记为运行并启动健康检查
4. 如果进程已退出 → 标记为真正失败

### 5. 健康检查增强

**现有功能**：
- 每 15 秒检查一次 `/system_stats`
- 连续失败 3 次才标记为失败
- 自动恢复机制

**新增优势**：
- 即使启动检测超时，健康检查也会持续监控
- 及时发现并恢复异常服务

## 核心改进代码

### 新增方法

1. **`_waitForFullEndpoints()`**：渐进式检测第二阶段
   - 检测完整端点
   - 进程存活检测
   - 宽容处理机制

2. **`_handleStartupFailure()`**：统一的失败处理
   - 清理资源
   - 杀死进程
   - 显示错误
   - 自动重启逻辑

### 改进方法

**`_waitForServerReady()`**：
- 分阶段检测
- 增加超时时间
- 改进日志输出

## 测试场景

### 场景 1：插件加载慢
- **原行为**：超时 → 杀死进程 → 失败
- **新行为**：超时 → 检测进程存活 → 标记运行 → 健康检查监控

### 场景 2：某些端点未响应
- **原行为**：超时 → 杀死进程 → 失败
- **新行为**：超时 → 基础端口检测 → 标记运行 → 健康检查监控

### 场景 3：进程真正崩溃
- **原行为**：超时 → 杀死进程 → 失败 ✓
- **新行为**：超时 → 检测进程退出 → 标记失败 ✓

### 场景 4：正常启动
- **原行为**：检测成功 → 标记运行 ✓
- **新行为**：快速检测 → 完整检测 → 标记运行 ✓

## 配置建议

### 1. 超时时间配置

根据插件数量调整：
- **少量插件（<20）**：180秒（3分钟）
- **中等插件（20-50）**：300秒（5分钟，默认）
- **大量插件（>50）**：600秒（10分钟）

在 `config.json` 中设置：
```json
{
  "server": {
    "timeout": 300000
  }
}
```

### 2. 自动重启配置

建议启用：
```json
{
  "server": {
    "autoRestart": true
  }
}
```

### 3. 监控日志

启动后关注以下日志：
```
[info] 启动超时设置: 300000ms
[info] 基础端口检测成功，继续完整端点检测...
[warn] HTTP端点检测超时，但进程仍在运行
[warn] 可能原因: 插件加载慢、某些自定义节点有问题、或端点响应慢
[info] ComfyUI启动成功，端口：8188
```
